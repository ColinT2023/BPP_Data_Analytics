Introduction¶

The Movies TMDB dataset was chosen for this analysis. The datset contains information about c.10,000 movies collated from The Movie Database (TMDB) from the years 1996-2015. The datset contains twenty-one columns including some interesting variables that will be included at various stages during this analysis which are as follows: budget; revenue, popularity, vote_average, cast and many more. I intend to discover meaningful insights and relationships within the data that will answer the following questions I have posed below.

Questions¶

  1. Is that data distributed noramlly across key variables?
  2. Movies with the highest and lowest revenue and what year were the Movies made?
  3. What are the Movies that contain the highest and the lowest budgets and what production companies made the film?
  4. What are the top five Movies with the highest profit?
  5. Plot Vote Average vs Profit to see if the most profitable movies matched the highest average votes?
  6. Is there a strong correlation between Revenue and Profit, and between Budget and Profit?
  7. What Genres feature most frequently within the data.
  8. Do the number of Movies released increase by year and how do budget, reveunue and profit trend over time.

Data Cleaning¶

When looking at the general proprties of the Movies dataset I found there are twenty one variables containg a mixture of datatypes that include integers, floats and objects/strings. The datseet also contains 10,866 rows although there are nine variables that contain missing values. Furthermore when looking at the first five rows of the data we can see that the cast, genres and production company variables contain data seperated by pipes. Upon first look I thought this may be an issue but it is perfectly reasonable to have multiple cast members, production companies and genres as movie attributes, although I will sepearate the genres attributes during the analysis to answer one of the queastions posed. The following steps will now occur to help clean the dataset:

Convert the release date variable into a date datatype.

Formatting float datatypes to suppress scientific notation and convert data into more readable figures.

Remove Movies that contain zero values for revenue and/or budget.

Remove the duplicate row (2090) that I found within the dataset.

Remove the columns that are not required for the purposes of this analysis i.e. tagline, homepage, keywords that contain considerable amount missing values, overview and imdb_id that are not required for this analysis, budget and revenue as the "overview & notes" section detailed within the datset options file states that the adjusted revenue and adjusted budget variables contains the revenue of the associated movie in terms of 2010 dollars,accounting for inflation over time which will give me a fair comparison between movies made in 1996 vs movies made in 2010 for example, hence the removal of the "revenue" and "budget" variables.

In relation to the nine variables that contain missing data; I will be removing some of the variables from the datsset as they are not required for this anlysis. Also as they are all of data type = Object/String, I will not be removing the rest of the missing values from the dataset as in doing so will remove valuable data from the key variables that will be used throughout this analysis.

Limitations¶

The Pearson correlation coefficient r, used to answer questions posed throughout this analysis, can take a range of values from +1 to -1. A value of 0 indicates that there is no association between the two variables. A value greater than zero indicates a positive relationship, If one variables increases so does the other. Additionall if a value is less than zero it indicates a negative relationship.

It is important to note that when determining if a relationship is “very strong” or “very weak” is subjective and correlation does not mean causation. Two variables can be strongly correlated, but that does not mean that changes in one variable causes the other variable to change. Or maybe an additional variable is actually the cause of changes within the first two.
